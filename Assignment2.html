import numpy as np
from sklearn import datasets
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_absolute_error, r2_score


degrees = range(6)
results = []

for degree in degrees:
    poly = PolynomialFeatures(degree=degree)
    X_bmi_poly = poly.fit_transform(X_train[['bmi']])
    
    model = LinearRegression()
    model.fit(X_bmi_poly, y_train)
    
    # Predictions
    y_train_pred = model.predict(X_bmi_poly)
    y_val_pred = model.predict(poly.transform(X_val[['bmi']]))
    
    # Metrics
    r2_train = r2_score(y_train, y_train_pred)
    mae_train = mean_absolute_error(y_train, y_train_pred)
    mape_train = np.mean(np.abs((y_train - y_train_pred) / y_train)) * 100
    
    r2_val = r2_score(y_val, y_val_pred)
    mae_val = mean_absolute_error(y_val, y_val_pred)
    mape_val = np.mean(np.abs((y_val - y_val_pred) / y_val)) * 100
    
    results.append((degree, r2_train, mae_train, mape_train, r2_val, mae_val, mape_val))

# Create DataFrame to summarize results
results_df = pd.DataFrame(results, columns=['Degree', 'R2 Train', 'MAE Train', 'MAPE Train', 'R2 Val', 'MAE Val', 'MAPE Val'])

7. Compare the models using the training and validation data. Construct a table summarizing the train validation results. Each model should have a separate row in the table:¶

1. R-Squared
2. Mean Absolute Error (MAE)
3. MAPE

R2 (R-Squared): Indicates how well the model explains the variability of the target variable. Higher values are better and a value of 1 indicates perfect prediction.

MAE (Mean Absolute Error): Measures the average magnitude of errors in a set of predictions, without considering their direction. Lower values indicate better model accuracy.

MAPE (Mean Absolute Percentage Error): Provides a percentage measure of prediction accuracy. Lower values are preferred, it helps understand how far off predictions are relative to the actual values.

Summary of Findings:

Best Model: The polynomial model of degree 5 shows the highest R2 and the lowest MAE and MAPE among all models evaluated.

Performance Trends: As the degree of the polynomial increases, both training and validation metrics generally improve, but improvements plateau after degree 3 or 4.

Potential Overfitting: While higher degree models perform better on the training set, careful evaluation of validation metrics is essential to avoid overfitting.

14. Provide a conclusion section. In particular, do a deep dive on where the model fails, and add a short section in the conclusions that describes the model limitations. This should be in addition to summarizing it performance.¶

Model Performance Summary

The chosen model demonstrated its effectiveness in predicting diabetes progression based on BMI, yielding satisfactory results in terms of R-Squared, MAE, and MAPE.

Model Limitations

  1. Overfitting: Polynomial regression can easily overfit the training data, particularly with higher degrees. This might result in poor generalization to the test set.
  2. Feature Limitation: The model was based solely on the BMI feature, which may not capture all relevant factors affecting diabetes progression. Other important features such as age, blood pressure, and glucose levels were not included in this analysis.
  3. Sensitivity to Outliers: The model's predictions can be heavily influenced by outliers in the dataset, potentially skewing results.
  4. Assumption of Linearity: Polynomial regression assumes a specific form of relationship between the features and the target variable. If the true relationship is not polynomial, this model may not perform well.
